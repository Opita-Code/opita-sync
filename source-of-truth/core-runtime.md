# Runtime del core

## Flujo base
contrato de intención/inspección -> acción de negocio -> tools/sistemas base

## Comportamiento ante intención incompleta
Mezcla ordenada:
- inspeccionar primero
- luego buscar memoria
- luego asumir y proponer
- preguntar solo si persiste ambigüedad crítica
- ejecutar solo cuando el contrato quede consistente

## Nivel de asunción
Configurable.

## Autonomía
Configurable por acción o tarea:
- automática
- con aprobación
- sin aprobación

## Tipos de resultado nativos del core
- plan
- inspección/análisis
- consulta
- propuesta de cambio
- ejecución
- actualización en sistemas
- aprobación/rechazo
- reporte/evidencia

## Workflows/agentes multistep
El sistema soportará ambos modos:
- predefinidos
- armados dinámicamente
