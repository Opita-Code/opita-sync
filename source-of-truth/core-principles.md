# Principios centrales

## Principios fijados
- producto final y profesional
- sin recortes artificiales por alcance
- el core debe capturar la mayor cantidad de datos útiles para análisis
- el producto nace como SaaS multi-tenant nativo
- lo que varía entre empresas se define al crear el tenant
- todo artefacto nace restringido por defecto
- si no se puede mostrar todo, el sistema devuelve salida redactada/parcial cuando esté permitido
- el sistema debe funcionar también para tenants de un solo usuario
- las decisiones técnicas secundarias se cierran por intención de negocio, sin gastar preguntas innecesarias

## Modelo mental del producto
intención -> contrato de intención/inspección -> inspección -> decisión -> acción gobernada -> evidencia
