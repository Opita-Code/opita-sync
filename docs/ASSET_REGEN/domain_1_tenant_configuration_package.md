# Domain 1 asset package — tenant configuration governance

## Domain

- `tenant configuration governance`

## What this package is for

Use this package when the story is about:

- what is enabled for a tenant
- capabilities
- connectors
- policy / approval / classification baselines
- tenant operability

## Primary audience

- admin tenant
- operator

## Core promise

Help teams govern **what exists and is enabled** for a tenant without losing preview, approval, evidence or recovery.

## Visual focus

- tenant bootstrap
- visible catalog
- connector projection
- baseline governance profiles
- governed preview and promotion

## Required message blocks

- tenant activation to operable
- enable / disable capabilities
- enable / disable connectors
- preview before promotion
- inspection and recovery when changes go wrong

## Assets this package should eventually produce

- landing section
- one-pager variation
- social / demo slide
- domain storyboard
- operator/admin walkthrough

## What this package should avoid

- broad interoperability claims
- generic AI platform framing
- access/delegation language as main theme
