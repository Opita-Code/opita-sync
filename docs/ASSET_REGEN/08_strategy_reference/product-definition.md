# Product definition

## What it is

**Opita Sync** is an **AI-first, governed control plane** built on top of **Opita Sync Framework (OSF)**.

It turns operational intent into **auditable, approvable, executable, inspectable and recoverable** changes and workflows over real systems.

AI leads the experience layer.
OSF remains the canonical authority for governance, execution, evidence and inspection.

## What it is not

Opita Sync is **not**:

- OSF itself
- a free agent
- a chatbot that directly executes actions
- a thin wrapper over APIs
- a universal engine without focus
- a UI detached from a governed kernel

## Relationship with OSF

The split is structural.

### OSF

OSF is the reusable framework/kernel.

It owns:

- canonical contracts
- compile path
- policy
- approvals
- runtime
- event log
- evidence
- inspection/recovery
- registry/resolution

### Opita Sync

Opita Sync is the product/control plane that consumes OSF.

It owns:

- AI-first experience
- tenant onboarding
- visible catalog
- admin/operator/approver surfaces
- derived realtime
- tenant activation
- ongoing product usage

## Guiding principle

**Opita Sync is AI-first in experience and OSF-first in authority.**

## Problem it solves

Organizations need to operate real systems with less friction, but without losing:

- control
- approvals
- traceability
- visibility
- inspection
- recovery

Opita Sync resolves that tension by combining an AI-first experience with a governed, auditable corridor.

## Value model

Opita Sync creates value in two stages:

1. **Tenant activation**
   - bootstrap until the tenant becomes operable
2. **Product usage**
   - ongoing use of the governed corridor

Commercially, the model is:

- tenant implementation / activation
- usage afterward

## Current product domains

### Domain 1

- `tenant configuration governance`

### Domain 2

- `tenant access and delegation governance`

## Initial governed corridor

`intake -> proposal -> preview -> governance -> execution -> inspection/recovery`

## Initial roles

- `admin tenant`
- `operator`
- `approver`

## Recommended positioning

Opita Sync should be positioned as:

**an AI-first governed control plane for operating business changes and workflows over real systems**

It should **not** be positioned as:

- a universal engine
- a generic enterprise AI platform
- a chatbot product
- the framework itself
